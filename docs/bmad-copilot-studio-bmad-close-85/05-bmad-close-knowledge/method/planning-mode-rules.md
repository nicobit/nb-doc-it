# Planning Mode Rules

Before generating the full handoff, classify the initiative as Quick, Standard, or Enterprise.

## Quick mode

Use for small enhancements with limited scope, few stakeholders, and low risk.

Required outputs:
- project brief
- capability summary
- epic candidates
- story candidates
- risks/questions
- handoff prompt

## Standard mode

Use for normal business initiatives with moderate scope and several stakeholders.

Required outputs:
- project brief
- capability map
- PRD draft
- epic candidates
- story candidates
- architecture context draft
- risks/assumptions/questions
- handoff readiness check
- handoff prompt

## Enterprise mode

Use for regulated, multi-system, high-risk, security-sensitive, or large initiatives.

Required outputs:
- all Standard outputs
- stronger evidence and traceability
- NFR table
- decision log
- dependency map
- governance notes
- higher scrutiny of assumptions and open questions

## Default

If the mode is unclear, use Standard.

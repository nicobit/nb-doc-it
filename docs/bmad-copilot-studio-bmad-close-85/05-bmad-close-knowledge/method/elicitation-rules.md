# Elicitation Rules

The agent must facilitate understanding, not merely summarize.

## Rules

- Identify the top unresolved questions before producing final artefacts.
- If the user asks for complete output anyway, proceed but mark uncertainty clearly.
- For every important requirement, classify it as Confirmed, Inferred, Assumption, or Open Question.
- Do not hide ambiguity.
- Prefer targeted questions over invented details.
- When assumptions are necessary, state why the assumption was made.
- Use confidence levels: High, Medium, Low.
- Include source evidence when available.
- Separate business decisions from technical assumptions.
- Highlight decisions required before BMAD implementation.

## Clarification priority

Ask or record questions in this order:

1. Business goal and success measure
2. Scope boundary
3. Primary users/personas
4. Critical business rules
5. Data/security/regulatory constraints
6. Dependencies and integration points
7. Release timing or sequencing constraints
8. Acceptance or readiness criteria

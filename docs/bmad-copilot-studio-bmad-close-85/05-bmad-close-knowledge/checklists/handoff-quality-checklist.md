# Handoff Quality Checklist

Before the package is considered ready for BMAD review, check:

- Business goal is clear.
- Scope and out-of-scope are documented.
- Primary personas/actors are identified.
- Capabilities are separated from features.
- PRD draft contains goals, users, scope, requirements, NFRs, risks, and questions.
- Epic candidates are outcome-based.
- Story candidates are labelled Candidate / Not implementation-ready.
- Architecture context is captured without inventing repo-specific implementation.
- Risks and assumptions are separated.
- Open questions are explicit.
- Handoff prompt for BMAD in VS Code is included.
- Final statement says BMAD must validate before coding.
